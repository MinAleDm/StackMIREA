## Теория

Следующий шаг после базовой валидации — извлечение данных через именованные группы и модульный подход к шаблонам.

## Код

```python title="practice_6_2_regex.py"
import re

log_pattern = re.compile(
    r"^(?P<date>\d{4}-\d{2}-\d{2})\s+"
    r"(?P<level>INFO|WARN|ERROR)\s+"
    r"(?P<message>.+)$"
)

line = "2026-02-10 ERROR Connection timeout"
match = log_pattern.match(line)
if match:
    print(match.groupdict())
```

## Разбор

Именованные группы (`?P<name>...`) делают результат парсинга более читаемым. Это удобно для конвейеров анализа логов и последующей сериализации данных.

Исходник практики: `pr_Python/pr_Python/pract6_2.ipynb`.

## Важные моменты

:::tip
Используйте именованные группы везде, где есть бизнес-сущности: дата, тип события, код ошибки.
:::

:::warning
Проверяйте шаблон на разных входных форматах, чтобы избежать ложных совпадений.
:::
