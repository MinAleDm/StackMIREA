## Теория

Связка UI-слоя, доменной логики и пользовательских ошибок. На этом этапе важно держать архитектуру модулей предсказуемой: классы с одной ответственностью, явные контракты и прозрачные зависимости.

## Код

```java title="Task17Example.java"
public final class Task17Example {
    private Task17Example() {
    }

    public static String run() {
        return "Task 17: module ready";
    }

    public static void main(String[] args) {
        System.out.println(run());
    }
}
```

## Разбор

Пример демонстрирует минимальный исполняемый модуль с изолированной точкой входа. В реальной задаче логика выносится в отдельные классы и тестируется независимо от `main`.

Исходники практики: `pr_Java/Task17/`.

## Важные моменты

:::tip
Дробите задачу на небольшие классы и методы: это ускоряет разработку и снижает количество дефектов.
:::

:::warning
Смешивание UI, бизнес-логики и работы с данными в одном классе быстро усложняет сопровождение.
:::
