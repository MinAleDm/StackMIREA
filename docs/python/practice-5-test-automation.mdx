## Теория

Автоматизированные тесты помогают фиксировать ожидаемое поведение кода и предотвращают регрессии после изменений.

## Код

```python title="test_discount.py"
def calc_discount(price: float, percent: float) -> float:
    if not 0 <= percent <= 100:
        raise ValueError("percent must be in [0, 100]")
    return round(price * (1 - percent / 100), 2)


def test_calc_discount() -> None:
    assert calc_discount(1000, 15) == 850
    assert calc_discount(500, 0) == 500
```

## Разбор

Функция имеет чёткий контракт: валидный диапазон процента и детерминированный результат округления. Тесты описывают нормальные сценарии и упрощают рефакторинг.

Исходник практики: `pr_Python/pr_Python/pract5.ipynb`.

## Важные моменты

:::tip
Пишите тесты на поведение, а не на детали реализации.
:::

:::warning
Неполная проверка граничных случаев часто приводит к ошибкам в продакшене.
:::
